2025-04-26 - Fix gfx1030 vector attention decode path and should_use_mmf fallback
Learning: The attention routing for RDNA2 (gfx1030) was inadvertently avoiding the highly optimal `BEST_FATTN_KERNEL_VEC` during small-batch decode (Q->ne[1] <= 4) simply because GQA optimizations applied to the KV cache, forcing a fallback to the slow tile kernel. Furthermore, `WARP_SIZE` was hardcoded to 32 inside the `ggml_cuda_should_use_mmf` fallback check, which could cause silent compilation or correctness issues on CDNA GPUs where wave size is 64. I also evaluated disabling CUDA graph capture entirely for AMD based on earlier assumptions but left it intact because of an external graph shim (`gfxGRAPH`) that achieves 100% cudagraph parity.
Action: Fixed the `warp_size` hardcoded bug in `ggml-cuda.cu` to correctly pull `ggml_cuda_info().devices[id].warp_size`. In `fattn.cu`, implemented a targeted override `if (GGML_CUDA_CC_IS_RDNA2(cc) && Q->ne[1] <= 4) return BEST_FATTN_KERNEL_VEC;` ensuring RDNA2 defaults to the vector kernel for the decode path regardless of GQA.
