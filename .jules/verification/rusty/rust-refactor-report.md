## Selected Candidate

- Path: `scripts/compare-logprobs.py`
- Current implementation: Single-threaded Python using `json.loads` per line, building lists in memory.
- Rust replacement: Pure Rust implementation using `serde_json`, streaming IO. Added as a binary to the existing `rusty` workspace.
- Reason selected: The compare step can be CPU-bound for large logs, and `serde` gives immediate 4x+ speedups parsing thousands of lines. Minimal integration required, no new network requests.

## Implementation Summary

I added a `compare_logprobs.rs` binary to the existing `rusty` crate (`rusty/src/bin/compare_logprobs.rs`). It takes identical CLI arguments for comparing files: `input1`, `input2`, `output`. It uses `serde_json` to parse the same format of logprobs generated by endpoints, extracting and formatting the tables in Rust, skipping empty lines securely, and outputting to both stdout and a markdown file matching Python output formatting exactly.

To fully replace the original functionality without removing the ability to dump via Python (which utilizes `requests`), I modified `scripts/compare-logprobs.py` so that when `compare` is called, it simply delegates to `rusty/target/release/compare_logprobs`.

## Compatibility Notes
