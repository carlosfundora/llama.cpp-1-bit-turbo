# Directory Structure Standard

## Purpose
Dictates the layout of the `docs/` folder.

## Required Directories
- `architecture/` - High-level system design.
- `subsystems/` - Canonical homes for major modules (e.g., backends).
- `standards/` - Coding, naming, and structure rules.
- `specifications/` - API and behavioral contracts.
- `reference/` - Low-level maps, env vars, configuration flags.
- `engineering-notes/` - Implementation details, findings, debugging.
- `data-maps/` - Schema maps, data flow.
- `migrations/` - Transition guides.
- `changelogs/` - Historical change records.
- `anti-patterns-and-best-practices/` - Operational high-value knowledge.
- `deployment/` - Deployment pipelines.
- `operations/` - Setup, manual tasks.
- `decisions/` - ADRs (Architecture Decision Records).
- `audits/` - Audit logs.
