# Documentation Style Standard

## Purpose
This standard dictates the style and formatting of documentation files in this repository.

## Applies To
All Markdown files in the `docs/` directory.

## Rules
- Prefer factual technical details over verbose narrative.
- Preserve specs, engineering notes, constraints, and architecture rationale.
- Never summarize away details needed for implementation.
- Files must use `kebab-case.md` formatting.

## Required Patterns
- Use Markdown headers for sections.
- Add frontmatter/header metadata block (see File Header Spec).
