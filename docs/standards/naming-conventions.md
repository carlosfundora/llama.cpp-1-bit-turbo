# Naming Conventions Standard

## Purpose
To ensure consistent naming across all documentation.

## Rules
- Documentation filenames must be descriptive, explicit, lowercase, and `kebab-case`.
- Understandable from the path alone.

## Preferred
- `runtime-boundaries.md`
- `module-boundaries.md`
- `database-schema-reference.md`
- `retrieval-pipeline-data-map.md`

## Avoid
- `notes.md`
- `misc.md`
- `stuff.md`
- `plan-final-v2.md`
- generic `index.md`
